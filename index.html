How it works

This page is the front door. The pipeline itself is a self-hosted n8n instance on Render, built and wired end-to-end — OAuth2, token flows, conditional branching and all.

Demo path (what you just ran) this page ──▶ POST /webhook/generate-article ──▶ Gemini writer agent ──▶ JSON back to this page Production path (runs on Google Sheets) new sheet row status=Pending ──▶ IF gate ──▶ Gemini writer ──▶ SEO analysis agent ──▶ sheet updated ──▶ markdown → HTML ──▶ approval email (send & wait) ──▶ approved? ──▶ WordPress REST publish ──▶ live URL written back to sheet
Human in the loop

Nothing goes live without a click. The workflow pauses on a real approval email with approve / decline actions before publishing.

Auth, the hard way

Self-hosted means no shortcuts: custom Google Cloud OAuth2 client for Sheets & Gmail, and a WordPress.com OAuth token exchange for publishing.

State lives in Sheets

Every article's lifecycle — Pending → Written → Published — plus its SEO report and live URL is tracked back into one Google Sheet.

Deployed, not local

n8n runs in Docker on Render (Singapore), tuned to fit a 512 MB instance. This site is static on Vercel and talks to it over CORS.
